S Spreedly Web SDK Callback Handler

Callback Handler

This endpoint is designed to receive server-to-server callbacks from Spreedly when a transaction status changes.

Note:

In a production environment, this would be a backend API endpoint (not an HTML page) that receives POST requests from Spreedly with transaction updates. The callback contains the full transaction details including the final status.

Expected Callback Data:

Spreedly sends a POST request with XML/JSON containing transaction details, including the transaction token, state (succeeded/failed), and payment method information.
